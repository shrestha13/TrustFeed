What happens on submission

① Publisher certificate is validated against the TrustFeed CA

② A 96-bit random nonce is generated and attached to the IOC

③ The IOC is serialized to canonical JSON (sorted keys, no whitespace)

④ Ed25519 signs the canonical bytes using the publisher's private key

⑤ The signed IOC is stored in the verified IOC store (SQLite)

⑥ A .tfb bundle is written to data/feeds/

Algorithm: Ed25519 signature over SHA-256 canonical bytes.
Every field — value, severity, nonce, timestamp — is part of the signed payload. Any modification invalidates the signature.
